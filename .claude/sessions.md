## 2026-05-18 — Runtime Bug Fixes (Map Load)

**Accomplished:** Diagnosed and fixed three bugs blocking map load. (1) Phaser 3.60+ no longer auto-calls `shutdown()` as a lifecycle method — fixed by adding `this.events.on('shutdown', this.shutdown, this)` in `create()` for MapSelectScene, GameScene, and UIScene. This was the root cause of #map-select not hiding when Play was clicked. (2) `Enemy` and (3) `Projectile` constructors were refactored to take `(scene, options)` when Entity Containers were introduced, but `_spawnEnemy` and `_updateTowers` in GameScene were never updated — both now pass `this` as the first arg. Map 1 (Outpost Sigma) confirmed loading in Playwright after fixes. Changes are not yet committed.
**Decisions made:** All three scenes need explicit `this.events.on('shutdown', ...)` registration — this is a Phaser 3.60+ breaking change that silently dropped the auto-binding. UIScene is a legacy parallel-scene architecture from Phase 3 that coexists with the Phase 5 GameScene; both need the fix.
**Where we left off:** Three files edited (MapSelectScene.js, GameScene.js, UIScene.js), changes uncommitted. Next: commit the fixes, then continue browser validation of PR #4 (story banners, star rating, phantom/titan visuals, localStorage).
